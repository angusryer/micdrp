# micdrp — Fastlane Lane Reference

Fastlane is invoked from `packages/client/` (the directory that contains
`ios/` and `android/`).  All signing material is consumed exclusively from
environment variables — **nothing is hard-coded in any Fastfile, Appfile, or
Matchfile**.

---

## Available Lanes

| Platform | Lane | What it does |
|---|---|---|
| iOS | `ios_beta` | Sync certs via Match, build with `gym`, upload to **TestFlight** (internal) |
| iOS | `ios_release` | Submit an approved TestFlight build to **App Store production** review |
| Android | `android_beta` | Assemble release AAB with Gradle, upload to **Play Store internal track** |
| Android | `android_release` | Promote the latest internal track build to **Play Store production** |

### Invocation examples

```sh
# Run from packages/client/
bundle exec fastlane ios ios_beta
bundle exec fastlane ios ios_release
bundle exec fastlane android android_beta
bundle exec fastlane android android_release
```

The wrapper shell scripts at the repo root (`scripts/release-ios.sh`,
`scripts/release-android.sh`) are the canonical entry points in CI.

## Required Environment Variables

All variables below **must be set as GitHub Actions secrets** (or locally
exported) before running any lane.  No default values are provided for
secrets — a missing variable will cause the lane to fail immediately with a
descriptive error.

### Common (both platforms)

| Variable | Description |
|---|---|
| `VERSION_NUMBER` | Semantic version string, e.g. `1.2.0`.  Falls back to `.env.production` if not set. |
| `BUILD_NUMBER` | Monotonically increasing integer build number.  Falls back to `.env.production` if not set. |
| `ENVFILE` | Path to the active env file (e.g. `packages/client/.env.production`). |

### iOS-specific

| Variable | Description |
|---|---|
| `APPLE_ID` | Apple ID email used to authenticate with App Store Connect / the Developer Portal. |
| `APPLE_TEAM_ID` | Apple Developer Portal Team ID (10-character string, e.g. `J3RCPBZHWM`). |
| `ITC_TEAM_ID` | App Store Connect Team ID (numeric string, e.g. `123456789`). |
| `IOS_BUNDLE_ID` | iOS bundle identifier, e.g. `io.ryer.micdrp`. |
| `MATCH_GIT_URL` | HTTPS or SSH URL of the private git repo storing Match certificates/profiles. |
| `MATCH_PASSWORD` | Passphrase used by Match to encrypt/decrypt certificates in the git repo. |
| `MATCH_KEYCHAIN_PASSWORD` | macOS keychain password used by Match when installing certificates on CI. |
| `MATCH_USERNAME` | Apple ID for Match (usually identical to `APPLE_ID`). |
| `ASC_API_KEY_PATH` | Filesystem path to the App Store Connect API key JSON file (created by `release-ios.sh` from `ASC_API_KEY_JSON`). |
| `ASC_API_KEY_JSON` | Raw JSON content of the App Store Connect API key (used by the CI workflow to write `ASC_API_KEY_PATH`). |
| `RELEASE_NOTES` | Optional changelog text surfaced in TestFlight. Defaults to a generic build string. |

### Android-specific

| Variable | Description |
|---|---|
| `ANDROID_PACKAGE_NAME` | Android application ID, e.g. `com.micdrp`. |
| `ANDROID_KEYSTORE_BASE64` | Base64-encoded release keystore file (`micdrp.keystore`).  The CI script decodes this and writes it to `ANDROID_KEYSTORE_PATH`. |
| `ANDROID_KEYSTORE_PATH` | Absolute path where the decoded keystore is written by the CI script (e.g. `/tmp/micdrp.keystore`). |
| `ANDROID_KEYSTORE_PASSWORD` | Password for the release keystore. |
| `ANDROID_KEY_ALIAS` | Key alias inside the keystore (`micdrpAndroidReleaseKey`). |
| `ANDROID_KEY_PASSWORD` | Password for the key alias. |
| `SUPPLY_JSON_KEY_DATA` | Full JSON content of the Google Play service-account key (the `json_key_data` form accepted by `supply`). |

### Files referenced by name (never committed with secret content)

The following files are **referenced by name** in build configuration but
their sensitive content is supplied at runtime:

| File | Purpose |
|---|---|
| `android/app/micdrp.keystore` | Android release keystore.  Written at CI time from `ANDROID_KEYSTORE_BASE64`. |
| `ios/micdrp/exportOptions.plist` | Xcode export options (method, team ID, provisioning profile names). Team ID is `J3RCPBZHWM` — update if the Apple account changes. |
| `ios/env.xcconfig` | Includes `tmp.xcconfig` generated by `react-native-config` at build time. |

## Setting Up Match (first time, on a Mac)

```sh
cd packages/client
# Create the private repo first, then:
bundle exec fastlane match init          # sets MATCH_GIT_URL in Matchfile (use ENV var, not literal)
bundle exec fastlane match appstore      # generate + encrypt + push certs/profiles
```

After the initial setup, CI runners use `readonly: true` and never modify the
Match repo.
